My show and tell is about implementing a dark mode toggle to theoretically any site using hooks and a hipster CSS property.

To implement a dark mode to a site, you need two things. First, you need a toggle to control dark mode and light mode. And second, you need to implement the actual styles for the dark mode.

Let's talk about the toggle first, which I implemented with hooks.

Before hooks

I lied to you a little bit by saying I implemented this thing with hooks + and the UI. Because, before this talk, I already implemented the dark mode in my site using internal state.

Let me first show you what the code looked like before hooks.

My site has a Layout component that wraps around everything. So the toggle goes into the Layout, right?

The Layout used to be a functional component, not anymore when I had to add an internal state for toggle.

The UI for the toggle also lives inside Layout. It implements a onClick handler which will setState for the toggle. It will also set the theme variable in local storage so you can leave the site and come back with your preferred theme.

Use hooks for the theme toggle

c1154dc

The first refactor was to rewrite the thing in hooks and move them to its own component. You can refer to this link for the code.

Here is the refactored component for the toggle.

I call useState with an initial state, which computes to the current theme if I see a local storage, otherwise use the default light theme.

The useState call returns an array which destructs to, one, a state variable, and two, a set state handler which I use to write my own toggleTheme handler. This toggleTheme handler is then passed to the onClick event for the div of the toggle.

Finally the write to local storage step is refactored into a useEffect hook. It will run whenever the theme changes. This is controlled by passing in the array with variable theme in to the second parameter to useEffect.

So we have our toggle lives in its own house together with its hooks, that's not bad? Now look at this file from a long distance and you'll notice the hooks logic comprises a cryptic block on this file. With hooks we don't have to keep that logic inside the component. We can take it out to its own file and import it to the components that need it.

Refactor to custom hook

Once again you can check out the commit here.

useTheme.js

This is one feature of hooks that people are excited about.

That separate function is commonly referred to as a "custom hook". I can let it return only the pieces of data or function it needs to share with the UI components that use it.

In this case, we return the theme variable, and the custom toggleTheme handler that we pass to divs as onClick event.

mix-blend-mode: difference
The idea is that there is a CSS property called mix-blend-mode that will ask you for a way to blend colors when graphics stack on each other. And then there is one such blending mode called difference that will blend the colors by taking their difference.

In CSS's standard RGB color space, white is all the light, black is no light which also means zero.

diff(white, white) = white - white = 0 = black

diff(white, black) = white - black = white - 0 = white

By taking the difference of my whole site against a full-screen white-colored div, the background that used to be white becomes black, and the foreground that used to be black becomes white.
