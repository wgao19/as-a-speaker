Compositing and blending specifies how graphics combine into one when you stack them together.

Compositing talks about how you combine shapes based on the alphas of each pixels.

And which (of the twelve) compositing method you are using defines if and how much of the pixel from your initial graphic and the compositing graphic it will take.

The method was initially defined by Porter Duff, who worked for Lucasfilm that later turned into Pixar.

Compositing is another interesting topic, but we will not cover that today.

CSS uses only simple alpha compositing (source over in Porter Duff).

How CSS does blending

Cr = (1 - ab) x Cs + ab x B(Cb, Cs)

  • Cr, Cs, Cb - resulting color, source color, backdrop color
  • blending modes specify B(Cb, Cs)
  • ab - alpha of backdrop

The result of the mixing function is modulated by the backdrop alpha. A fully opaque backdrop allows the mixing function to be fully realized. A transparent backdrop will cause the final result to be a weighted average between the source color and mixed color with the weight controlled by the backdrop alpha.

To understand what is done by the blending funciton "multiply", we need to understand the CSS color space.

More on Color Spaces by Bartosz Ciechanowski

And in particular, we need to understand the RGB color model.

The RGB color model is additive in the sense that the three light beams are added together.

When all three beams are at the strongest intensity, we get white. When the beams are less intense but are equal in their intensity, we get a point on the greyscale. When they have different intensity, we get a colorized hue.

We can represent RGB color model is based on Cartesian coordinate system.

Consider the unit cube, where red, blue, and green sit on the ‚Äú1‚Äù ends of the axis. The ‚Äú1‚Äù means that color beam at its highest intensity, and ‚Äú0‚Äù means no light at all.

CSS uses the

sRGB color space

rgb(100%, 65%, 2%)

rgb(66, 53, 94)

The CSS color module specifies that with each channel, 0% represents minimum value, and 100% represents the maximum value.

Alternatively, we can use a number, where 0 again represents the minimum value for the color channel, but 255 represents the maximum.

"multiply"

What it actually means is how the two color beams weaken each other. And since each one of the three color beams are either left unchanged (if multiplied with 1) or weakened, the result is at least as dark as either of the two colors.

It follows that multiplying any color with black gives you black, like you turn off the lights. And multiplying any color white white gives you the original color, like no weakening at all.

The resultant color is always at least as dark as either of the two constituent colors. Multiplying any color with black produces black; multiplying with white leaves the original color unchanged.

When the source color is bright, it brightens the backdrop. 
            Proof. 

            Hard light case Cs > 0.5:

            B(Cb, Cs) = Screen(Cb, 2 * Cs - 1)
                      = 2 * (Cb + Cs - Cb * Cs) - 1 

            Comparing with Cb, want to know if 2(Cb + Cs - Cb * Cs) - 1 - Cb > 0

            LHS = Cb + 2 * Cs - 2 * Cb * Cs - 1
                = Cb * (1 - 2 * Cs) - (1 - 2 * Cs)
                = (Cb - 1) * (1 - 2 * Cs)

            Since Cs > 0.5, 2 * Cs > 1, therefore 1 - 2 * Cs < 0;
            Cb - 1 <= 0 because Cb <= 1

            Hence LHS >= 0 and the conclusion is it will always brighten the backdrop.

            QED.

            LHS = (Cs - 1) * (1 - 2 * Cb)

            Now we know that Cs - 1 <= 0 because Cs <= 1.
            However, whether the other term is >0 or not is unknown.

            If Cb > 0.5, then 2 * Cb > 1 and 1 - 2 * Cb < 0. 
            In this case the final product is >= 0, which implies it brightens the source.
            Otherwise, the result is reversed.
