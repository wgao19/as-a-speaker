What is Gatsby

Static Site Generator

Content pages are pre-built instead of requested from and generated by a server.

Prior arts and similar projects include Jekyll, Hugo, or Next.

It may or may not involve an opinionated front end framework as UI renderer. Like Gatsby is one with React, Gridstone with Vuejs.

Some are opinionated regarding use case, for example Docusaurus is one targeting specifically to documentations. And it provides versioned docs, multi language support et cetera and is very popular among open source projects such as Redux.

based on React + GraphQL

  • Layout, templates, and reusable UI parts are all React components.
  • GraphQL manages all the data and asset queries

Up and running

Environment dependencies

  • git
  • node.js version >= 8
  • gatsby-cli

gatsby-cli 


$ npm install -g gatsby-cli
$ gatsby-cli --version

The command gatsby new comes from the package Gatsby CLI. It is intended to be installed globally. Once we‚Äôve correctly installed the Gatsby CLI, we should be able to run the command gatsby with the --version modifier to verify that it‚Äôs properly installed.

Gatsby CLI commands 


$ gatsby new name-of-your-site url-to-starter # defaults to gatsby-starter-default
$ gatsby develop  # starts a hot reload server
$ gatsby build  # may be run by your deployment service
$ gatsby serve  # serves production build
$ gatsby info  # info about your system and your package
$ gatsby clean

The `gatsby new` creates a new gatsby project. It should be run followed by the name of your project site, then followed by the starter you use. It defaults to use gatsby-starter-default if you do not provide the last argument.

`gatsby develop` starts a hot reload development server that you may use for development.

`gatsby build` builds your site targeting production. This is also the default build command used by deployment services such as Netlify. We‚Äôll cover that later.

`gatsby serve` starts a server that serves production build after you‚Äôve built it.

`gatsby info` prints out information about your system as well as your package.

And finally `gatsby clean` is useful when some issue (mainly caching) arrises in your development.

Would like to bring to your attention to this file: `gatsby-config.js`, which contains the configurations for our gatsby site generation. This is commonly provided by starters.

siteMetadata (object): When you want to reuse common pieces of data across the site (for example, your site title), you can store that data in `siteMetadata` which can be later picked up by your GraphQL query.

And in our case our `siteMetaData` contains our site url, and the `rssMetaData` that is being consumed by the `gatsby-plugin-feed` plugin, which generates an RSS feed for our site. You may visit the RSS feed by going to `/rss.xml` in the browser.

With Gatsby Advanced Starter it puts some meta information in the directory data/SiteConfig.js, and imports directly in their React components, passed down via props, and is not queried by GraphQL. Also works.

plugins (array): Another essential configuration is the plugins. Some plugins may need only to be listed by name, while others may take options (see the docs for individual plugins).

There are a few more options that you may read about in Gatsby‚Äôs config documentation:

Typography.js
A powerful toolkit for building websites with beautiful design.

Gatsby‚Äôs creator Kyle has created a TypographyJS library (and a large number of util libraries around that). What TypographyJS does is it provides a toolkit for us to define the typography design of our site.

TypographyJS is a monorepo package and it publishes alongside of it a repertoire of themes that you may use directly. You may see those themes on their demo website.

You may as well define your own typography. How it works is that you define some basics to your site‚Äôs typography. It will help you sort out (or lets you customize) the font sizing, line spacing, etc, using a mechanism called vertical rhythm.

For today‚Äôs workshop, we are going to create our own Typographyjs theme.

`overrideStyles`

Imperative API for directly adding to or overriding auto-generated styles. It‚Äôs called with a Vertical Rhythm object, the options object, and the algorithmically generated styles. 

overrideStyles: () => ({
  "*": {
    background: "#222",
    color: "#eee"
  },
  "@media and (min-width: 35rem)": {
    maxWidth: "35rem"
  },
  "blockquote > :last-child": {
    marginBottom: 0,
  },
})

You should specify a function of three parameters for TypographyJS to call. Your function should return your desired stylesheet in an object, where the field names are the selectors, and the field values are the CSS value object (kebab converted to camelCase according to React convention).

This allows us to specify highly customized CSS styles. Since the field names are simply CSS selectors, it supports:

  • media queries
  • pseudo element selectors

Furthermore, it provides us with a few util functions that we may use to declaratively write our typography styles. Let‚Äôs talk about Vertical Rhythm.
